# Implementation Plan: v3.0.x Core Mesh Protocols

This plan details the steps for implementing the foundational communication and discovery protocols for AgentMesh v3.0.

## 1. v3.0.1 — Agent Registry (Discovery)
- **Goal**: Enable agents to find each other on the Nostr network based on capabilities.
- **Tasks**:
  - [ ] Define `AgentCapability` Pydantic model in `packages/agentmesh-core`.
  - [ ] Update `NostrAgent` in `packages/agentmesh-relay` to support publishing and querying capability events (Kind 30311 or custom).
  - [ ] Implement a discovery cache in `MeshOrchestrator`.

## 2. v3.0.2 — Standardized AgentMessage (Communication)
- **Goal**: Unified task-oriented messaging between agents.
- **Tasks**:
  - [ ] Define `AgentMessage` Pydantic model.
  - [ ] Implement message signing and verification using Nostr keys.
  - [ ] Update `BaseAgent` to include an `on_message` handler.

## 3. v3.0.3 — Task Agent (Orchestration)
- **Goal**: Abstract complex task delegation.
- **Tasks**:
  - [ ] Create `TaskAgent` in `apps/podcast-generator/podcast_generator/agents/`.
  - [ ] Implement routing logic to delegate podcast generation steps to Content, Storage, and Network agents.

## Verification
- Unit tests for message serialization and signing.
- Integration test with a mock Nostr relay to verify discovery and message passing.
